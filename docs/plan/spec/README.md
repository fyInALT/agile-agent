# Spec Folder Policy

This directory contains implementation-facing engineering specs for `agile-agent`.

## Language Policy

All spec documents under `agile-agent/docs/plan/spec/` must be written in English.

Rationale:

- Specs in this folder are intended to be implementation-ready.
- English keeps terminology stable across code, comments, issue tracking, and future automation.

## Scope

Documents here should focus on:

- execution plans
- engineering checklists
- runtime behavior
- file/module targets
- acceptance criteria
- non-goals

Product strategy, roadmap, and backlog planning can live outside this folder.
